## Codebase Patterns
- Use `uv run pytest` from repo root
- FundamentalsProvider protocol at `stock_analysis/ai/fundamentals/provider.py` — follow same `@runtime_checkable` + Protocol pattern for new protocols
- CompositeProvider (YFinance + EDGAR fallback) lives in `stock_analysis/ai/fundamentals/` — delegate to it for financials, don't duplicate
- Test mocking: use `monkeypatch.setattr(yfinance, "Ticker", lambda sym: mock)` for yfinance isolation
- Sentinel pattern for test helpers: use `_MISSING = object()` default args to distinguish None from "use default"
- New subpackages need `__init__.py`; diagnostics pattern (`tests/diagnostics/`) has one, `tests/ai/` does not
- MarketDataProvider module: `stock_analysis/market_data/` — protocol + YFinanceProvider live here
- `helpers.py` module-level `_default_provider = YFinanceProvider()` — patch via `monkeypatch.setattr(helpers_mod, "_default_provider", mock)` in tests
- `YFinanceProvider` has extra non-protocol methods: `get_raw_ticker`, `get_quarterly_balance_sheet`, `get_quarterly_financials`, `get_tickers_obj`
- `get_raw_ticker(ticker)` → returns `yf.Ticker` object; use when caller needs raw ticker API (history with interval, dividends, info, etc.)
- `get_tickers_obj(symbols)` → returns `yf.Tickers` batch object; used by fetch_stocks_data / fetch_stocks_data_alt
- `StockValuation(ticker, provider=None)` in `valuation_methods.py` — provider replaces raw `yf.Ticker` arg; uses `get_raw_ticker` internally
- `PaidMarketDataStub` in `market_data/paid_stub.py` — raises NotImplementedError for future FMP/Polygon/Stooq adapters; see `market_data/README.md`
- Consumer modules (`stock_finder`, `sector_analysis`, `sp500_analysis`, `analyze_stock_data`, `linear_reg_sig_days`) use `_default_provider` pattern with optional `provider` kwarg
