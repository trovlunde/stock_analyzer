## Codebase Patterns
- Run tests with `uv run pytest` from repository root
- Package management: `uv add` / `uv sync` — never pip install into the project venv manually
- Tests should use fixtures/mocks; avoid live market data in CI
- Fundamental classification cache contract: success paths return `(X, y)` DataFrames
- CLI entry points: `stock-analysis-daily`, `stock-analysis-evaluate` (see pyproject.toml)
- `prepare_classification_data_cache` accepts `_cache_dir=None` — pass `tmp_path` in tests to avoid touching real cache
- `prepare_classification_data` returns 3-tuple `(features_df, targets_df, returns_df)`; cache helper unpacks all 3 then returns only `(features_df, targets_df)`
